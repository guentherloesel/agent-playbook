# Task Handoff Checklist

Before handing work back to the user:

- Confirm the requested change is complete and scoped to the task.
- Run the most relevant validation available in the repository.
- Report any validation that could not be run and why.
- Check `git status` and call out staged, unstaged, and untracked files.
- Summarize the changed files and the behavior affected.
- Mention any remaining risk, follow-up, or assumption that matters for review.
